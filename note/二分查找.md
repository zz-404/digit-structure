我们把这个方法叫做“二分答案”。顾名思义，它用二分的方法枚举答案，并且枚举时判断这个答案是否可行。但是，二分并不是在所有情况下都是可用的，使用二分需要满足两个条件。一个是有界，一个是单调。

二分答案应该是在一个单调闭区间上进行的。也就是说，二分答案最后得到的答案应该是一个确定值，而不是像搜索那样会出现多解。二分一般用来解决最优解问题。刚才我们说单调性，那么这个单调性应该体现在哪里呢？

如果题目规定了有“最大值最小”或者“最小值最大”的东西，那么这个东西应该就满足二分答案的有界性（显然）和单调性（能看出来）。

二分时需要注意边界left right的取值
避免除零错误
细节注意二分区间为 [1,L] 而非 [0,L]，否则会出现除以 0 RE 的情况。

一定要看好我们设置的变量，到底满足条件时谁在扩展谁在保留
例如：
```
 while (left <= right) {
        int mid = (left + right) / 2;
        int need = 0;
        for (int i = 1; i < n; ++i) {
            int dist = arr[i] - arr[i - 1];
            need += (dist - 1) / mid;   // 新增路标数
        }
        if (need > k) {          // mid 太小，不可行
            left = mid + 1;
        } else {                 // mid 可行，尝试更小值
            right = mid - 1;
        }
    }
    // 循环结束时，left 是第一个可行的 mid（即最小最大间隔）
    cout << left << '\n';

为什么输出 left 而不是 right？
当 mid 可行时，我们执行 right = mid - 1，把可行值排除在外，继续向左搜索。

最终 left 会停在最小的可行值，而 right 停在它左边的值（不可行或小于答案）。

所以答案应为 left。
```
